Snowplow vs Adobe Analytics

The following guide outlines the main reasons why you should use Snowplow Insights to power your analytics stack over Adobe Analytics.
It’s worth noting that Adobe is an end-to-end web analytics solution (you integrate the trackers on your website(s), and you get the reports in their user interface). In contrast, Snowplow Insights is not an end-to-end analytics solution: it is a data collection and processing platform. It is built from the ground up to support event collection across any platform, with web and mobile just the two most popular. Email, displays ads, push notifications, call center, help desk, in-store, Smart Car, Smart Home, Internet of Things are some of the other supported platforms.
Snowplow Insights is not a SaaS product - each user has their own data pipeline running on their own AWS account. This gives each user complete freedom to define what events they want to record, how those events should be structured and how they want to process them. Snowplow Insights delivers the data directly to the locations where it is most useful: into our client’s own data warehouse and own Unified Log (real-time event stream in Amazon Kinesis).

Define Your Own Custom Events & Entities

Adobe Analytics has a fixed schema: every company running Adobe Analytics, be it an publisher, online roleplaying game, social network etc. will have the same core table with the same columns. Whilst eVars and Props allow the user to add their own custom metrics and dimensions to the core table structure – the end result is a long list of numbered variables appended to the core table structure which don’t clearly describe the interactions they are supposed to represent. With Snowplow Insights, we don’t assume that any two businesses want to track the same events and entities nor do we assume that they want to answer the same questions. An online publisher may want to understand what content of theirs is most viral whilst a recipe company might want to see which cuisines their users enjoy cooking. In practice this means that setting up Snowplow takes longer as from the ground up, you decide:

  1. The questions you want to answer.
  2. The data you want to track.
  3. How you want to use the data to answer those questions.
The benefit in running through this process is that, in the end, you have a reporting setup that answers your most important questions, in your business’ language.

Out-of-the-box fields

There are several fields which are available in Snowplow upon setup which just aren’t available in Adobe Analytics, here are some examples:

  • IP Adresses of the end user
  • Page ping i.e heartbeat events. These include a number of data points which allow you to calculate metrics such as how far down a page a user has scrolled (which in turn allows you to calculate how much content is read) as well as how long users spend on particular content items.
  • Performance timing data i.e data on how fast pages take to load.
  • IDFA (Open, Apple and Android) / IDFV (Apple).

Marketing Integrations

Snowplow has a large number of integrations with third party marketing and data providers whilst Adobe offers none:

  • Adjust for mobile app attribution
  • Augur for machine-fingerprinting
  • AWS Cloudfront for log ingestion
  • CallRail for call tracking
  • Clearbit for enriching user information
  • Mailchimp for email marketing data
  • Mandrill for transactional email data
  • Optimizely for A/B testing
  • OpenWeatherMap for weather data
  • PagerDuty for devops alerts
  • Pingdom for uptime alerts
  • SendGrid for transactional email tracking
  • Urban Airship for mobile marketing

Data latency

It can take over 24 for new data to land in the Adobe reporting interface. In contrast, Snowplow users can run their pipelines hourly, loading data into Redshift with a 2 hour latency. Users of the Snowplow real-time pipeline have the full event stream available to consume in Kinesis and Elastic in seconds.

Cost

With Adobe Analytics, it’s easy to rack up a bill of over $100,000 per year. Running Snowplow Insights is an order of magnitude cheaper.
